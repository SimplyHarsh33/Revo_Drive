📌 SECTION 1 — EVERYONE MUST KNOW

1.1 What is DriveSafe AI?

DriveSafe AI is a real-time driver monitoring system that uses Edge AI — meaning all AI processing happens directly in the user's browser using the device's own GPU. No video is ever sent to any server, protecting driver privacy. When a dangerous event (drowsiness, phone use, yawning) is detected, it is logged to a backend database via a REST API.

Key selling point: Runs entirely in-browser (zero cloud AI cost), but still persists telemetry to a central server.

1.4 What is Edge AI?

Edge AI means running the artificial intelligence model on the same device that collects the data, instead of sending data to a remote server (cloud).

  • Cloud AI: Camera → Upload video to server → Server runs AI → Send result back → Slow, costly, privacy risk
  • Edge AI: Camera → Run AI in browser → Instant result → Only log events to server ✅

1.5 Common Viva Questions

Q: What is the main purpose of this project?

A: To monitor driver alertness in real-time using AI that runs directly in the browser (Edge AI), and log dangerous events (drowsiness, phone use, yawning) to a central database for post-trip analysis.

Q: Why did you choose Edge AI instead of sending video to a server?

A: Three reasons — (1) Real-time speed: local inference is ~instant, no network delay. (2) Privacy: no driver video ever leaves the device. (3) Cost: cloud inference APIs charge per call; WebAssembly and WebGL are free.

Q: What does the backend actually store?

A: It stores two things — a Session (a driving trip, with start/end time) and Events (individual alerts like "DROWSY at 14:32:05 with EAR score 0.18").

Q: What is CORS and why is it needed?

A: Cross-Origin Resource Sharing. The frontend runs on port 5173, the backend on port 8000. Browsers block cross-port requests by default. FastAPI's CORSMiddleware explicitly permits this.

🔧 SECTION 2 — BACKEND (Harsh)

2.1 Technology: FastAPI

FastAPI is a modern Python web framework for building REST APIs. It is based on Python type hints, automatically generates interactive docs (Swagger UI), and uses an async-capable ASGI server (Uvicorn).

2.3 How `db.py` Works

SQLALCHEMY_DATABASE_URL = "sqlite:///./revodrive.db"

engine = create_engine(SQLALCHEMY_DATABASE_URL, connect_args={"check_same_thread": False})
# "check_same_thread": False is needed because SQLite was designed for single-thread
# use, but FastAPI handles multiple requests simultaneously (multi-threaded).

SessionLocal = sessionmaker(autocommit=False, autoflush=False, bind=engine)
# SessionLocal is a "factory" — every time you call SessionLocal(), you get a
# new database session (like a transaction).

2.5 What is an ORM?

ORM = Object Relational Mapper. Instead of writing raw SQL (INSERT INTO sessions...), you write standard Python classes, and SQLAlchemy translates them into SQL automatically.

2.7 Viva Q&A — Backend

Q: What is the difference between a Pydantic model and a SQLAlchemy model?

A: A Pydantic model defines the shape of data coming IN and going OUT of the API (JSON validation). A SQLAlchemy model defines the database TABLE structure.

Q: What does `Depends(get_db)` do?

A: It's FastAPI's Dependency Injection. It automatically passes a live database session to each endpoint and closes it afterwards to prevent memory leaks.

Q: What is Uvicorn?

A: Uvicorn is an ASGI (Asynchronous Server Gateway Interface) server. It's what actually runs the FastAPI app and listens for HTTP requests.

🎨 SECTION 3 — FRONTEND UI & ALARMS (Jivit)

3.1 Technology: React + Vite + TypeScript

React uses a Virtual DOM — React updates only the changed parts of the screen, not the whole page. Vite acts as a super-fast build tool using ES Modules. TypeScript catches type errors at compile-time.

3.3 How the Landing Boot Sequence Works

Without the LandingScreen.tsx boot sequence, the browser requests camera permission immediately on page load, which looks unprofessional and can fail before models load. Our boot sequence delays to load "MediaPipe WASM Cores" and "TensorFlow.js WebGL" visually.

3.4 How the Web Speech API Works

const utterance = new SpeechSynthesisUtterance(
    "Alert! Driver drowsiness detected. Please pull over!"
);
window.speechSynthesis.speak(utterance);

This is a built-in browser API (zero cost). We added a 3-second setTimeout cooldown to prevent voices from endlessly stacking.

3.7 Viva Q&A — Frontend

Q: What is the Virtual DOM in React?

A: React keeps a lightweight in-memory copy of the DOM. When state changes, it computes the difference (diff) between old and new state, then only updates the changed DOM nodes instead of re-rendering everything.

Q: What is the difference between useState and useRef?

A: `useState` triggers a whole UI re-render when the value changes. `useRef` stores a value that persists without triggering a re-render. We used `useRef` to fix the double-increment yawn bug in React 18 Strict Mode.

🧠 SECTION 4 — ML FACEMESH CORE (Mallika)

4.1 Technology: MediaPipe FaceMesh

MediaPipe FaceMesh detects 468 3D facial landmarks in real-time via WebAssembly (WASM), running C++ speed directly inside the browser without plugins.

4.2 Eye Aspect Ratio (EAR)

EAR = (Vertical_Distance_1 + Vertical_Distance_2) / (2.0 * Horizontal_Distance)

Based on Soukupova & Cech 2016. If the EAR drops below ~0.25 on average for both eyes, the driver is flagged as drowsy.

4.6 Viva Q&A — ML FaceMesh

Q: What is the EAR formula and what does it measure?

A: It measures the openness of the eye using 6 facial landmarks per eye. EAR drops toward 0 as the eye closes.

Q: What is WebAssembly (WASM)?

A: A binary format that runs in browsers at near-native speed. It allows MediaPipe's C++ ML engine to run in JS.

📦 SECTION 5 — ML OBJECT DETECTION (Divyanshu)

5.1 Technology: TensorFlow.js + COCO-SSD

TensorFlow.js runs directly in the browser via WebGL (GPU acceleration). COCO-SSD ("Single Shot MultiBox Detector") detects 80 object classes, including the `cell phone`.

5.3 The 2-Second Hold Logic

A 1-frame "flash" of a cell phone might be a false positive. We only trigger the alarm if the `cell phone` is continuously detected for 2+ seconds using timestamp tracking.

5.4 Frame Throttling

Running ML inference 60 times a second crashes browsers. We use a modulo operation frameCount % 30 === 0 inside requestAnimationFrame to run detection merely 1-2 times per second.

5.6 Viva Q&A — Object Detection

Q: What is COCO-SSD and how does it work?

A: SSD (Single Shot Detector) detects objects and their bounding box positions in a single neural network forward pass, making it fast enough for live video.

Q: What is WebGL and why does TensorFlow.js use it?

A: WebGL is a browser API for GPU graphics. TensorFlow.js uses it as a "backend" to run massive matrix math on the GPU rather than the slower CPU.

🐳 SECTION 6 — DEVOPS (Hemant)

6.1 Technology: Docker & Compose

Docker packages our app and all its dependencies into an isolated container that runs identically on any machine, eliminating "works on my machine" issues.

6.2 Understanding `docker-compose.yml`

services:
  backend:
    build: .
    ports:
      - "8000:8000"
    depends_on:
      - db

It binds our host's port 8000 to the container's 8000, and ensures the PostgreSQL database spins up before the FastAPI server.

6.6 Viva Q&A — DevOps

Q: What is the difference between an image and a container?

A: An image is a read-only blueprint. A container is a specific running instance of that image.

Q: Why use PostgreSQL in Docker instead of SQLite directly?

A: SQLite is a single file and struggles with concurrent reads/writes under heavy traffic. PostgreSQL is a full client-server database built for production.
